Was ist eine URL?
Der Begriff

URL(Uniform Resource Locater)

Die URL ist die Adresse, welche oben im Browser eingegeben wird, sie wird auch Internetadresse genannt.
Ein Beispiel für eine URL ist youtube.com.
Die URL wird dich dann auf die gewünschte Seite bringen.
Damit dies funktioniert, werden URLs jeweils nur einmal vergeben, sie sind also eindeutig.

Was ist das DNS?
Der Begriff

DNS (Domain Name System)

Das DNS sorgt dafür, dass deine eingegebene URL in eine IP-Adresse umgewandelt wird.
Die IP-Adresse wird benötigt, damit dein Computer weiß an wen er seine Websiteanfrage senden muss.
Denn der Server ist durch die IP eindeutig erkennbar.
Also wird www.example.com zu 192.0.2.44.
Diese schickt er zu deinem PC zurück und dein PC stellt dann eine Verbindung zu dieser IP her.
Der Server mit dieser IP, schickt dir dann die angeforderte Website, wenn du also youtube.com aufgerufen hast, wird dir ein Video angezeigt.

Man kann sich das DNS also wie ein Telefonbuch vorstellen, du kennst den Namen einer Person und guckst nach unter welcher Nummer du die Person erreichen kannst.
Erst danach rufst du die Person unter dieser Nummer an.

Ein HTML Sprunglink
Der Begriff

Bei einem HTML Sprunglink kann zu einer gewissen Stelle im Text gesprungen werden.
Die Stelle zu der gesprungen wird, ist festgelegt durch eine ID.

Bsp:
<div id=“zielort“>Ziel</div>

Danach muss festgelegt werden von wo zu dieser Stelle gesprungen werden soll, also zum Beispiel von einem Inhaltsverzeichnis.
Dazu benutzt man den href und schreibt in die Anführungszeichen #id.
Für die id wird die id des Zielobjekts eingefügt.

<a href="#zielort">zum Anfang der Seite</a>

Wenn man im Body-Tag noch „scroll-behavior: smooth;“ einfügt, sorgt dies für einen scrollenden Übergand anstatt eines Sprunges.
(Ausprobieren)


<style>
  html {
    Scroll-behavior: smooth;
  }
</style>

Die Anwendung Wget
Wget

Wget ist ein freies Konsolentool, um Dateien aus dem Internet herunterzuladen.
Es wurde von GNU Projekt im Jahre 1996 entwickelt und wird seither immer noch benutzt.
Auf den meisten Unix basierten Betriebssystemen ist Wget meist vorinstalliert, jedoch gibt es auch eine Version für Windows.

Zur Zeit unterstützt Wget die Protokolle: HTTP, HTTPS und FTP.
Wget wurde dazu entwickelt, große Mengen an Daten herunterzuladen, auch wenn es zwischendurch Netzwerkfehler gibt.
Außerdem kann es im Hintergrund laufen, auch wenn man nicht eingeloggt ist.

Die Besonderheit von Wget ist die hohe Einstellungsmöglichkeit, mit der man genau beschreiben kann, wie Wget arbeiten soll.
Es lässt sich z.B. so einstellen, dass Wget ganze Websites herunterlädt, um so eine Offlineversion der Website zu erstellen.
Wget kann Links auf der Webseite folgen, und so rekursiv die Website downloaden.

Installation
Auf Unix Distributionen ist Wget, wie schon erwähnt, meist vorinstalliert.
Falls es jedoch nicht installiert ist, kann man es mit folgendem Kommando installieren (UNIX):

sudo apt install wget


Auf Windows kann man sich hier eine ZIP-Datei mit Binaries herunterladen, welche sich dann nach dem entpacken ausführen lassen.

Gebrauch
Um das Programm zu benutzen, muss man es über eine Konsole ausführen.
Bei Unix basierten Betriebssystemen benutzt man z.B. das Terminal und bei Windows z.B. die Eingabeaufforderung bzw. die PowerShell.

Wenn man auf Windows arbeitet, ist es wichtig, die wget.exe am Anfang des Befehls zu erwähnen, damit dein PC weiß, welche Datei er ausführen soll.
Es empfielt sich daher, den Befehl entweder im gleichen Ordner auszuführen, wo die wget.exe Datei vorhanden ist oder diesen Ordner in den PATH zu schreiben, damit sie von überall gefunden werden kann.

Syntax
Der Grundlegende Syntax von Wget lautet:

wget [option]... [URL]...

[option]... : hier beschreibt man, wie Wget herunterladen soll. Eine Liste der Optionen erhält mit mit dem Kommando wget --help

[URL]... : Liste von URLs, deren Inhalt Wget herunterladen soll.

Beispiele
Um z.B. das Logo der Website des Goethe Gymnasiums Lichterfelde herunterzuladen, gibt man folgendes ein:

wget http://goethe-gymnasium-lichterfelde.de/files/goethe-logo_header.png

Dieses eher simple Beispiel lässt sich vergleichen, wie wenn man im Webbrowser auf das Bild rechtsklickt und auf 'Bild speichern' klickt.

Wenn man nun die ganze Website des Goethe Gymnasiums herunterladen wollte, könnte man folgendes eingeben:

wget -r http://goethe-gymnasium-lichterfelde.de/

In dem Fall würde die Option -r bewirken, dass alle Inhalte rekursiv heruntergeladen werden, dass heißt wenn ein Link gefunden wird, folgt das Programm diesem und sucht wieder nach Links und lädt alle Inhalte herunter.

Dies würde alle Inhalte, die irgendwie auf der Website hinterlegt sind herunterladen. (zur Zeit ~1267 Dateien mit zusammen ~446 MB Größe)
Man erhält eine völlig offline kompatible Website, die man sich jetzt lokal auf dem Computer ansehen kann.
Dieser Schritt ist ähnlich zu dem, was der Webbrowser macht, wenn man die Website aufruft.
Der einzige Unterscheid ist, dass der Webbrowser sich nicht gleich die ganze Website herunterlädt, sonder nur die Inhalte, die für die momentane Seite relevant sind, lädt.
